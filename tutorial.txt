WHAT THIS PROGRAM DOES
----------------------
The tool reads a Base64 ciphertext from a JSON file, decodes it to raw bytes,
then builds a large pool of candidate plaintexts:

  1) Heuristics — lore/Treyarch keyword lists, single-byte XOR on decoded
     bytes, Base64 alphabet rotations, Atbash on Base64, Vigenère and Beaufort
     on the outer Base64 string, reversed ciphertext, XOR on Base64 ASCII,
     and layered transforms (nested Base64, zlib, hex-as-ASCII, nibble swap,
     repeating XOR, RC4, optional full single-byte XOR on layer buffers).

  2) Extended probes — classical ciphers on the Latin-1 view of the payload
     (Beaufort/Vigenère on A–Z and on A–Za–z, Atbash, Caesar shifts, ROT13,
     rail-fence), cumulative-XOR unroll on bytes, and OpenSSL block decrypt
     attempts (AES, 3DES, RC2/Blowfish when your Node/OpenSSL exposes them)
     including decrypting after reversing ciphertext bytes.

  3) Automatic brute force (time-limited by default) — after heuristics,
     repeating XOR with keys from [a-zA-Z0-9] up to a max length, then
     Vigenère plus Beaufort on the Base64 ciphertext with the same alphabet,
     until readable English is found or the wall clock budget expires.

It scores outputs with English-oriented heuristics (ASCII, letters, dictionary
hits, chi-squared, quadgrams) and applies a strict readability gate for a
“solved” result.

Important: this program searches and ranks candidates. It does not guarantee
that any real-world ciphertext will decode to readable English.


PREREQUISITES
-------------
  • Node.js version 18 or newer (see package.json "engines").
  • npm (comes with Node).

Check your version:

  node -v


FIRST-TIME SETUP
----------------
  1) Open a terminal in the project folder (where package.json lives).

  2) Install dependencies:

       npm install

  3) Build TypeScript (most npm scripts run build for you):

       npm run build


INPUT FILE FORMAT
-----------------
The default file is cipherbase.json in the project root. It must contain a
JSON object with a string field named "cipher" holding the Base64 ciphertext.
Whitespace in the string is stripped when decoding.

Example:

  {
    "cipher": "YOUR_BASE64_STRING_HERE"
  }

You may add other fields (e.g. "mainkey"); the solver only requires "cipher".

Use another path:

  node dist/index.js --file path/to/your.json


MAIN npm SCRIPTS
----------------
Run from the project directory. Scripts use cross-env for Windows/macOS/Linux.

  npm run build
      Compiles TypeScript to dist/

  npm run solve   (same as npm start)
      Default “wide search” preset: large extended-keyword list, deep nested
      Base64 (12), large layer buffer cap, full layer XOR/RC4 key lists (see
      package.json for exact GIANT_* numbers). Brute key lengths default to
      XOR 4 / Vigenère+Beaufort 3 and 10-minute wall time unless you override
      environment variables.

  npm run solve:quick
      Fast loop: brute 2/2, 3-minute budget, smaller extended + layer caps.

  npm run solve:full
      Brute 4/4, 10-minute budget, same wide extended + layer preset as solve.

  npm run solve:30m
  npm run solve:1h
  npm run solve:6h
  npm run solve:24h
      Brute 5/5, time budget in the name, wide extended preset, tuned progress
      logging for long runs.

  npm run solve:unlimited
      GIANT_MAX_MS=0 — brute phases run until you stop (Ctrl+C). Interrupt
      prints last known brute progress.

  npm run dump
      npm run build, then --dump-all: list scored heuristic candidates without
      the automatic brute loop.

  npm run help
      Builds and prints CLI flags and environment variable names.

Worker threads for brute force default to your CPU logical core count. Set
GIANT_THREADS (e.g. 24) if you want to cap or raise parallelism.


COMMAND-LINE OPTIONS
----------------------
  -f, --file <path>     JSON with { "cipher": "<base64>" }
  --dump-all            All candidates, no auto brute (legacy)
  --min-score <n>       With --dump-all: score floor
  -n, --limit <n>       With --dump-all: max rows
  --jsonl               With --dump-all: one JSON object per line
  -h, --help            Help text


ENVIRONMENT VARIABLES (SUMMARY)
--------------------------------
Full list: npm run help

Brute phases:
  GIANT_MAX_MS            Wall time for XOR + Vig/Beaufort brute (ms); 0 = unlimited
  GIANT_BRUTE_XOR_LEN     Max repeating XOR key length [a-zA-Z0-9]
  GIANT_BRUTE_VIG_LEN     Max Vig/Beaufort key length on Base64 layer
  GIANT_PROGRESS_EVERY    XOR progress log interval (0 = off)
  GIANT_THREADS           Worker threads (default: CPU count)
  GIANT_EXTRA_KEYS        Extra keys for heuristics (comma or semicolon)

Extended probes (disable to speed up):
  GIANT_EXTENDED_PROBES       0 = skip classical + block + cumulative-XOR bundle
  GIANT_CLASSICAL_PROBES      0 = skip classical letter ciphers
  GIANT_BLOCK_CIPHER_PROBES   0 = skip OpenSSL block decrypt tries
  GIANT_EXTENDED_KEYWORDS     Cap on merged keyword list
  GIANT_CLASSICAL_KEYS        Cap for classical keys only
  GIANT_BLOCK_CIPHER_KEYS     Cap for block-cipher passphrases
  GIANT_BLOCK_CIPHER_MAX      Max successful block decrypts per direction

Layered search:
  GIANT_NESTED_B64, GIANT_LAYER_MAX_BUFS, GIANT_LAYER_XORREP, GIANT_LAYER_RC4,
  GIANT_LAYER_XOR256, GIANT_LAYER_XOR256_MAXBUFS

Logging:
  GIANT_WORD_LOG          Dictionary word log path (default giant-word-hits.log)


WHAT YOU WILL SEE (typical run)
--------------------------------
  • Yellow “cryptanalysis running” banner with keyspace notes and time budget.
  • On an interactive terminal (TTY): live brute panel — pipeline progress,
    keys tried/left, phase budget countdown, ETA (during XOR/Vigenère brute).
  • After completion: hero, input strip, outer Base64 payload analysis (entropy,
    hex preview, file signatures, nested-format hints), hex pattern scan
    (annotated dump, XOR printable hints, structural notes), analytics
    dashboard, execution pipeline, dictionary token scan, path to word log,
    best candidate / readability cards.

  The payload and hex panels depend only on the decoded bytes of "cipher";
  they do not change run-to-run unless the ciphertext changes.

  giant-word-hits.log — session metadata, ciphertext sections, dictionary
  token positions, and appended payload / hex analysis text.


TIPS
----
  • Use solve:quick first; scale up to solve, solve:full, or timed long runs.
  • Brute wall time applies only to XOR + Vigenère/Beaufort phases, not to the
    entire process from process start.
  • GIANT_EXTRA_KEYS adds community or map-specific guesses without code edits.
  • For GitHub-style documentation, see README.md in the repo root.
